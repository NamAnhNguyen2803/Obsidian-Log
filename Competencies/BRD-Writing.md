# BRD Writing

> Năng lực viết Business Requirements Document đủ chất lượng để Dev implement và QA test mà không cần hỏi lại.

---

## Định nghĩa năng lực

Khả năng phân tích requirement, cấu trúc thành BRD rõ ràng — bao gồm business flows, business rules, UI/UX notes, integration points, và acceptance criteria.

---

## Sub-skills

### BRD Structure
- [x] Document info (version, status, stakeholders)
- [x] Objectives & scope
- [x] User flows (happy path + alternative paths)
- [x] Business rules (numbered, testable)
- [x] Data fields table (field, type, validation, required)
- [x] Integration points (API calls, external systems)
- [ ] Non-functional requirements
- [ ] Acceptance criteria (Gherkin format)

### SOW-based BRD (sản phẩm triển khai)
- [x] Contract → SOW → BRD flow
- [x] Identify change vs no-change từ base product
- [ ] Market customization matrix (VN vs Timor vs ...)

### Review & Iteration
- [x] Self-review checklist (8 lenses)
- [ ] Stakeholder sign-off flow
- [ ] Version history management
